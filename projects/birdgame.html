About the Project

Birdgame is an Unreal Engine 5 game I have been designing and building independently. It combines custom level design with Blueprint-driven gameplay mechanics and original game logic implemented in both C++ and UE5's visual scripting system.

The project is an ongoing creative and technical challenge — it is where I explore game-systems thinking, 3D environment design, and the Unreal toolchain at a deeper level than coursework provides.

What I Built

  • Custom level environment designed from the ground up in UE5's World Partition editor
  • Player character class with movement mechanics implemented in C++
  • Enemy behaviour and interaction logic built using Blueprint Visual Scripting
  • Game state management (win/lose conditions, scene transitions)
  • Lighting, atmosphere, and environmental storytelling passes
Approach: I use C++ for performance-critical or reusable gameplay systems and Blueprints for rapid iteration on level-specific logic — matching how professional UE5 teams typically split responsibilities.

Challenges & What I Learned

Learning the UE5 ecosystem

Unreal Engine has a steep learning curve. I invested significant time understanding how the engine manages assets, how the renderer works, and how C++ classes integrate with the Blueprint inheritance hierarchy.

Managing binary assets in version control

Large asset files are genuinely difficult to manage with standard Git. Using Git LFS locally taught me firsthand why binary-heavy projects require a different approach to version control — a real-world constraint that many developers don't encounter until they hit it.

Scope management as a solo developer

Without a team to set boundaries, scope creep is a constant risk. I'm practising defining clear milestones and shipping small, playable builds rather than waiting for a "complete" product.
