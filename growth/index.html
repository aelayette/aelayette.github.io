Portfolio Development

Draft → Final: What Changed

My draft portfolio was a single-page site with project cards that named the apps but didn't explain the technology, the decisions, or what I actually contributed. It also had no project evidence or evidence of learning besides links to my GitHub. The feedback from my instructor made it clear that a list of names is not enough evidence.

The current version rethinks the entire structure around case studies: each project now has its own page with a tech stack panel, explicit contributions, an evidence gallery with captioned screenshots and videos where applicable, as well as a next steps section which includes my gaps in knowledge. The goal is to show real proof, not just presence.

Key improvement: Moving from "I built a thing" to "Here is what I built, why I built it that way, what was hard, and what I would do differently."

Learning

Skills I've Grown

Team Collaboration

Working on S.I.R.E. with a four-person team taught me that process matters as much as code. Clear PR templates, consistent standups, and shared branching conventions made collaboration possible at a pace where individual heroics would have failed.

Android Development

Building the Elite Bake Shop App from scratch in Jetpack Compose — including managing local SQLite data without an ORM — gave me a working mental model of how Android's data and UI layers fit together.

Real-time Web Systems

Implementing Socket.IO with namespaced rooms for S.I.R.E. pushed me to think about server state, event broadcasting, and the challenges of keeping multiple clients in sync — concepts that don't appear in basic web dev courses.

Technical Documentation

Writing the S.I.R.E. project poster and communications plan taught me that good documentation is a technical skill, not an afterthought. Explaining architecture in plain language for a non-technical audience forced me to understand it more deeply myself.

Game Development Fundamentals

Learning Unreal Engine 5 independently — starting with a structured course then continuing on my own — built genuine resilience for navigating complex, poorly-documented tools and large, unfamiliar codebases.

Local Database Design

Designing the SQLite schema for the Elite Bake Shop App — writing raw SQL for schema creation, insertion, and querying — gave me a hands-on understanding of relational database design and what ORMs are actually doing under the hood.

Honest Self-Assessment

Gaps I've Identified

Automated Testing

I've done manual testing and written informal test plans, but I haven't built a systematic automated test suite (unit, integration, or end-to-end) for any project yet.

Plan: Add JUnit tests to the Elite Bake Shop App data layer as a first step. Learn Jest for the S.I.R.E. frontend. Work through a testing fundamentals course (JUnit/Mockito for Android; Vitest or Jest for web).

Production Deployment

I've used GitHub Pages and Docker in coursework, but I haven't deployed a full production application with proper environment management, secrets handling, or reliability monitoring.

Plan: Deploy a personal project to a cloud provider (Railway, Render, or similar) with proper environment config and uptime visibility.

CI/CD Pipelines

I've used GitHub Actions at a basic level but haven't configured a full CI/CD pipeline with automated build, test, and deploy stages.

Plan: Set up a GitHub Actions workflow for a personal project that runs linting and tests on every PR and auto-deploys on merge to main.
